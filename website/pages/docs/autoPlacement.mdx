# autoPlacement

The `autoPlacement` middleware chooses the `placement`
automatically (the one with most space available on checked
axes).

## Usage

```js
import {computePosition, autoPlacement} from '@floating-ui/dom';

computePosition(referenceEl, floatingEl, {
  middleware: [autoPlacement()],
});
```

## Options

`autoPlacement` accepts an object of options.

### alignment

Without options, `autoPlacement` will choose any of the
`BasePlacement`s which fit best, i.e. `top`, `right`, `bottom`,
or `left`.

By specifying an alignment, it will choose those aligned
placements.

```js
autoPlacement({
  // top-start, right-start, bottom-start, left-start
  alignment: 'start',
});
```

### crossAxis

Describes whether to check the `crossAxis` when choosing the
ideal placement.

```js
autoPlacement({
  crossAxis: true, // false by default
});
```

### allowedPlacements

Describes the placements which are allowed to be chosen.

```js
autoPlacement({
  // 'right' and 'left' won't be chosen
  allowedPlacements: ['top', 'bottom'],
});
```

### autoAlignment

When `alignment` is specified, this describes whether to
automatically choose placements with the opposite alignment if
they fit better.

```js
autoPlacement({
  alignment: 'start',
  // Won't also choose 'end' alignments if those fit better
  autoAlignment: false, // true by default
});
```
